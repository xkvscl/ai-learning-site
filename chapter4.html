⚙️ 第四章：Agent 核心原理

深入理解 AI Agent 的核心机制，掌握 ReAct、工具调用、记忆管理等关键技术

⏱ 预计学习时间：2周 📊 难度：进阶

🤖 4.1 什么是 AI Agent？

AI Agent（人工智能代理）是一种能够自主感知环境、制定计划、调用工具并执行任务的智能系统。与普通 AI 对话不同，Agent 能主动做事，而不只是回答问题。

🔄 Agent 工作循环

👁️
感知（Perceive）

接收用户输入和外部环境信息（工具返回结果）

💭
推理（Reason）

分析任务，决定下一步行动（调用哪个工具）

⚡
行动（Act）

调用工具获取结果（搜索、计算、代码执行等）

🔁
循环

持续循环直到任务完成

Agent 与普通 AI 的核心区别

对比项普通 AI（ChatGPT）AI Agent
工作模式一问一答自主规划 + 循环执行
工具调用❌ 不能✅ 可以（搜索/计算/写代码）
多步骤任务❌ 一次性回答✅ 分步骤自主完成
访问互联网❌ 知识有限✅ 可以（配合搜索工具）
执行代码❌ 只能展示✅ 真实执行
记忆仅当前对话✅ 可持久化存储

🧠 4.2 ReAct 框架 — 推理与行动的结合

ReAct = Reasoning（推理）+ Acting（行动）。这是最经典的 Agent 推理框架，核心思想是：思考 → 行动 → 观察 → 思考 → ... → 完成

# ReAct 循环的伪代码 def react_loop(task): observation = "" # 初始观察（用户输入） for step in range(max_steps): # 1. Thought（思考）：根据当前情况决定做什么 thought = llm.think( context=conversation_history, observation=observation, available_tools=tools ) # 2. Action（行动）：执行思考决定的动作 if thought.action == "search": observation = search(thought.query) elif thought.action == "calculate": observation = calculate(thought.expression) elif thought.action == "finish": return thought.result # 任务完成 # 3. 将思考和结果加入历史，循环

ReAct 示例：让 Agent 查天气并回答

# 用户问题："北京今天适合出门吗？" # Agent 思考过程（可见）： [Thought] 我需要知道北京的天气情况才能回答。 可以调用天气工具获取实时信息。 [Action] call_tool("get_weather", city="北京") [Observation] 北京：温度18°C，多云，空气质量良好 [Thought] 根据天气信息，北京今天天气不错，适宜出门。 可以给用户一个完整的建议。 [Final Answer] 北京今天天气良好（18°C，多云），非常适合出门活动！ # 用户问题："帮我查下 GPT-4 的发布时间" [Thought] 这是一个事实性问题，需要搜索最新信息。 [Action] call_tool("search", query="GPT-4 发布时间") [Observation] GPT-4 于 2023 年 3 月 14 日由 OpenAI 发布。 [Thought] 找到了答案，可以直接回复用户。 [Final Answer] GPT-4 于 2023 年 3 月 14 日发布，距今约 X 年。
💡 为什么 ReAct 有效？ReAct 让 AI 模拟人类的思考过程——先想想该怎么做，再动手，然后根据结果调整下一步。这种"慢思考"比直接回答复杂问题更准确！

🛠️ 4.3 工具调用 — Agent 的超能力

工具是 Agent 与外部世界交互的桥梁。通过调用工具，Agent 可以做普通 AI 完全做不到的事情。

常见工具类型

工具类型功能示例
🔍 搜索工具访问互联网获取最新信息Bing Search、Google
💻 代码执行运行代码获取计算结果Python REOL、Bash
📄 文件操作读写本地文件Read/Write File
🌐 网页访问抓取网页内容Browser、Scraper
📧 邮件工具发送/读取邮件Email Send/Gmail
📊 数据库查询/操作数据库SQL Query
🔧 自定义工具根据需求开发任何你能想到的功能

Function Calling — 定义自己的工具

# 用 OpenAI Function Calling 定义一个"计算器"工具 import openai, json tools = [ { "name": "calculate", "description": "执行数学计算", "parameters": { "type": "object", "properties": { "expression": { "type": "string", "description": "数学表达式，如 '2+3*5' 或 '10**2'" } }, "required": ["expression"] } }, { "name": "get_weather", "description": "查询城市天气", "parameters": { "type": "object", "properties": { "city": {"type": "string", "description": "城市名称"} }, "required": ["city"] } } ] response = openai.ChatCompletion.create( model="gpt-4o", messages=[{"role": "user", "content": "计算 128 * 256，再查下上海天气"}], tools=[{"type": "function", "function": t} for t in tools] ) # 解析 AI 返回的工具调用 for choice in response.choices: msg = choice.message if msg.tool_calls: for tool_call in msg.tool_calls: name = tool_call.function.name args = json.loads(tool_call.function.arguments) print(f"调用工具: {name}, 参数: {args}")
🎯 核心思想：你可以给 Agent 定义任意工具！Agent 会在需要时自动判断该调用哪个工具。工具的定义方式就是描述"工具是做什么的"和"需要什么参数"。

🧠 4.4 记忆管理 — Agent 的"大脑"

普通 AI 只记得当前对话的历史。Agent 需要更强大的记忆系统，才能在多轮交互中保持连贯。

三种记忆类型

记忆类型存储内容生命周期类比
🟢 短期记忆当前对话上下文单次会话工作记忆
🟡 长期记忆重要历史信息跨会话持久化笔记本/日记
🔴 情景记忆经验教训/偏好永久知识库

实现一个简单的记忆系统

import json from datetime import datetime class AgentMemory: def __init__(self): self.short_term = [] # 短期：当前会话 self.long_term = [] # 长期：以文件形式存储 def add(self, role, content, memory_type="short"): """添加记忆""" entry = { "role": role, "content": content, "time": datetime.now().isoformat() } if memory_type == "short": self.short_term.append(entry) else: self.long_term.append(entry) self._save_to_file() def get_context(self, max_turns=10): """获取完整上下文（短期 + 重要的长期）""" recent = self.short_term[-max_turns:] important = [m for m in self.long_term if m.get("important")] return recent + important def summarize_and_store(self): """定期将短期记忆压缩存储到长期""" if len(self.short_term) > 20: summary_prompt = "请总结以下对话的要点：\n" + \ "\n".join([f"{m['role']}: {m['content']}" for m in self.short_term]) # 调用 LLM 生成摘要 summary = "（此处调用LLM生成摘要）" self.add("system", f"对话摘要: {summary}", memory_type="long") self.short_term.clear() def _save_to_file(self): """持久化到本地文件""" with open("memory.json", "w", encoding="utf-8") as f: json.dump(self.long_term, f, ensure_ascii=False, indent=2) # 使用 memory = AgentMemory() memory.add("user", "我叫小明，是一名Python初学者") memory.add("assistant", "好的小明！很高兴认识你，我会记住你的名字。") context = memory.get_context() print(f"当前上下文: {len(context)} 条记忆")
💡 RAG（检索增强生成）是大模型记忆系统的重要技术——把大量知识存入向量数据库，Agent 需要时检索相关记忆再生成回答。第五章会详细讲。

📋 4.5 规划能力 — 制定行动方案

Agent 需要能够把复杂任务拆解为多个子步骤，并按计划执行。

任务分解（Task Decomposition）

# 简单任务分解 user_task = "帮我分析 AAPL 股票并给出投资建议" # Agent 自动拆解为子任务： subtasks = [ "1. 搜索 AAPL 最新股价和财务数据", "2. 分析近3年营收增长趋势", "3. 对比同行业竞争对手", "4. 综合分析给出投资建议" ] # 或者让 AI 自动分解 decomposition_prompt = f""" 请将以下任务分解为具体步骤： 任务：{user_task} 请列出步骤，每个步骤： - 明确要做什么 - 可能需要什么工具或信息 """ # result = llm.generate(decomposition_prompt) # print(result)

React 规划模式 vs 一次性规划

模式特点适用场景
🔁 React（边做边想）每步根据结果调整，灵活探索性强、结果不确定的任务
📝 Plan-then-Act（先想后做）先制定完整计划，再执行目标明确、可预见的任务
🔄 混合模式先制定计划，执行中动态调整复杂真实项目

👥 4.6 多 Agent 协作

多个 Agent 各司其职、协作完成复杂任务，比单个 Agent 更强大。

单 Agent vs 多 Agent

单 Agent = 一个人完成所有工作（厨师 + 服务员 + 收银）
多 Agent = 团队分工协作（厨师做菜、服务员上菜、收银结账）

多 Agent 协作示例：智能客服系统

# 多 Agent 协作架构示例 agents = { "router": { "role": "路由 Agent", "description": "理解用户问题，判断转给哪个专员", "prompt": "你是客服路由，根据问题类型转接..."}, "tech_support": { "role": "技术支持 Agent", "description": "处理技术问题", "prompt": "你是技术支持工程师..."}, "sales": { "role": "销售 Agent", "description": "处理购买咨询和报价", "prompt": "你是销售顾问..."}, "supervisor": { "role": "监督 Agent", "description": "审核答复质量，确保专业", "prompt": "你是质检员，检查以下回答是否合适..." } } def multi_agent_workflow(user_question): # 1. 路由 Agent 判断问题类型 route = router_agent.decide(user_question) # 2. 转给对应的专业 Agent if route == "technical": answer = tech_support_agent.answer(user_question) else: answer = sales_agent.answer(user_question) # 3. 监督 Agent 审核 reviewed = supervisor_agent.check(answer) return reviewed

📏 4.7 Agent 评估方法

如何衡量一个 Agent 的好坏？以下是常用评估维度。

维度指标说明
🎯 任务完成率成功率 / 步骤数Agent 能独立完成任务吗？用了多少步？
⏱️ 效率Token 消耗 / 执行时间用了多少资源？
🔍 准确性答案正确率调用工具的参数正确吗？结果解读正确吗？
🛡️ 安全性有害输出率有没有绕过安全限制？
🤝 稳定性多次运行一致性同样问题，结果稳定吗？
💬 交互体验用户满意度回答是否有条理、易理解？
