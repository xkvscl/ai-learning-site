🔗 第三章：LLM API 调用

学会用代码调用大语言模型，构建你的第一个 AI 应用

⏱ 预计学习时间：1周 📊 难度：进阶

🔑 3.1 获取 API Key

API Key 是调用 AI 服务的"通行证"，每个服务都需要先获取自己的 Key。

OpenAI API Key 申请

1. 访问 platform.openai.com/api-keys
2. 登录/注册账号
3. 点击 "Create new secret key"
4. 复制并保存 Key（只会显示一次！）

Claude API Key 申请（推荐免费用）

1. 访问 console.anthropic.com
2. 注册账号（有免费额度）
3. 在 API Keys 页面创建 Key

⚠️ 安全警告：API Key = 银行卡密码！
- 不要写在代码里
- 不要提交到 GitHub
- 不要分享给任何人
- 使用环境变量存储

安全存储 API Key（环境变量）

# Windows PowerShell 设置临时环境变量 $env:OPENAI_API_KEY = "sk-xxxxxxx" # Windows CMD 设置 set OPENAI_API_KEY=sk-xxxxxxx # 或创建 .env 文件（需要 python-dotenv） # .env 文件内容： # OPENAI_API_KEY=sk-xxxxxxx # ANTHROPIC_API_KEY=sk-ant-xxxxxxx
💡 推荐做法：使用 python-dotenv 库从 .env 文件读取，这样 .env 不会上传到 GitHub。

💬 3.2 OpenAI API 完整指南

安装 SDK

# 安装 OpenAI Python SDK pip install openai # 安装 dotenv（用于环境变量管理） pip install python-dotenv

基础对话调用

import openai import os from dotenv import load_dotenv # 从 .env 读取 API Key load_dotenv() openai.api_key = os.getenv("OPENAI_API_KEY") # 最简单的对话调用 response = openai.ChatCompletion.create( model="gpt-4o-mini", # 指定模型 messages=[ {"role": "system", "content": "你是一个有帮助的助手"}, {"role": "user", "content": "什么是Python？"} ], temperature=0.7 # 随机性（0-1） ) # 打印回复 reply = response.choices[0].message.content print(reply) # 查看 token 用量 print(f"本次消耗 Token: {response.usage.total_tokens}")

流式输出（逐字显示）

import openai response = openai.ChatCompletion.create( model="gpt-4o-mini", messages=[{"role": "user", "content": "用一句话解释什么是AI"}], stream=True # 开启流式输出 ) # 逐块接收响应（打字机效果） full_reply = "" for chunk in response: if chunk.choices[0].delta.content: text = chunk.choices[0].delta.content full_reply += text print(text, end="", flush=True) print()

function calling（函数调用）

Function Calling 让 AI 能够调用你定义的函数，是 AI Agent 的核心技术！

import openai response = openai.ChatCompletion.create( model="gpt-4o", messages=[{"role": "user", "content": "北京今天多少度？"}], functions=[ { "name": "get_weather", "description": "查询城市天气", "parameters": { "type": "object", "properties": { "city": { "type": "string", "description": "城市名称，如北京、上海" } }, "required": ["city"] } } ] ) message = response.choices[0].message if message.function_call: func_name = message.function_call.name args = message.function_call.arguments print(f"AI想调用函数: {func_name}") print(f"参数: {args}")

💭 3.4 多轮对话与上下文

让 AI 记住对话历史，实现真正的多轮交流。

import openai # 维护完整对话历史 messages = [ {"role": "system", "content": "你是一个Python编程助教"}] def ask(user_message): """发送消息并获取回复""" messages.append({"role": "user", "content": user_message}) response = openai.ChatCompletion.create( model="gpt-4o-mini", messages=messages ) reply = response.choices[0].message.content messages.append({"role": "assistant", "content": reply}) return reply # 对话轮次 1 print("用户: Python中列表和元组有什么区别？") print("AI:", ask("Python中列表和元组有什么区别？")) # 对话轮次 2（AI记住了之前的上下文） print("\n用户: 那字典呢？") print("AI:", ask("那字典呢？")) # AI知道你在问Python的数据结构 # 查看完整历史 print("\n=== 对话历史 ===") for msg in messages: print(f"[{msg['role']}]: {msg['content'][:50]}...")
⚠️ 上下文长度限制：每个模型有最大 Token 数限制（上下文窗口）。超过会截断旧消息。GPT-4o 有 128K tokens，约10万汉字，一般够用。Claude 3.5 Sonnet 有 200K tokens，更长。

🛡️ 3.5 错误处理与稳定性

调用 API 可能遇到网络问题、额度用尽等错误，良好的错误处理让程序更稳定。

import openai import time def chat_with_retry(prompt, max_retries=3): """带重试机制的对话函数""" for attempt in range(max_retries): try: response = openai.ChatCompletion.create( model="gpt-4o-mini", messages=[{"role": "user", "content": prompt}] ) return response.choices[0].message.content except openai.error.RateLimitError: # 触发了速率限制，等待后重试 wait_time = 2 ** attempt # 指数退避 print(f"触发限流，等待 {wait_time} 秒...") time.sleep(wait_time) except openai.error.APIError as e: # API错误，可能服务器端问题 print(f"API错误: {e}") time.sleep(5) except openai.error.AuthenticationError: # API Key无效 print("错误：API Key 无效！请检查配置") break return "抱歉，服务暂时不可用，请稍后再试。" # 使用 result = chat_with_retry("你好") print(result)

常见 API 错误及解决方案

错误类型原因解决方法
RateLimitError请求过快加延迟/减少并发
AuthenticationErrorKey无效/过期检查/重新获取API Key
InvalidRequestError参数错误检查model/messages格式
APIConnectionError网络问题检查网络/加代理
timeout请求超时增加timeout参数

💰 3.6 API 成本控制

AI API 按 Token 计费，学会控制成本很重要。

成本优化技巧

# 1. 使用更小更快的模型 # 简单任务用 mini 模型，省钱90% response = openai.ChatCompletion.create( model="gpt-4o-mini", # 比 gpt-4o 便宜50倍！ messages=[{"role": "user", "content": "1+1等于几"}] ) # 2. 控制输出长度 response = openai.ChatCompletion.create( model="gpt-4o-mini", messages=[{"role": "user", "content": "总结这篇文章"}], max_tokens=100 # 限制最大输出100 tokens ) # 3. 只传必要的历史消息（节省输入token） def trim_history(messages, max_turns=10): """只保留最近N轮对话""" return messages[:1] + messages[-max_turns*2:] # system + 最近N轮 # 4. 使用缓存（Anthropic Claude 支持） # 把不变的系统提示缓存起来，省输入token
💡 建议：开发阶段用 GPT-4o-mini 或 Claude Haiku，等功能稳定后再换大模型。
